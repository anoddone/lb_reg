 rx_p2p_mpd_ns0x0meanPathDelay
valid and value in ns.
The peer-to-peer mechanism delivers meanPathDelay for each ingress
port. This needs to be added to the
Sync
packets correction field before the packet is sent out on
egress port. Thus, the egress port might add any of the ingress ports'
'meanPathDelay'.
The value to be added at the egress port should correspond to
the ingress port on which the
Sync
packet has arrived.

Bit 30Indicates meanPathDelay is valid.
Bits 0 to 29meanPathDelay value in nanosecond.
Bit 31reserved.
